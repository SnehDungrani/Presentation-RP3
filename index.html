~/Inter-Act/05-architecture

Architecture · current state (hybrid)

Centralized intent, decentralized execution

click to zoom ⤢
centralized (strategic) decentralized (tactical + recovery) actor-to-actor on the bus
Hybrid by design. The operator sets a goal; a central interpreter and announcer turn it into an auction on the shared bus. From there, execution, actor-led fault detection, and recovery run across the actors - the recovery claim is authored by another actor, not the centre. Each actor is its own container; the 0.5B narrator only writes operator bubbles.

~/Inter-Act/06-coordination

How coordination works

Bidding decides. The LLM only advises.

1. announce 2. all AGVs bid 3. claim winner
mechanism Actors bid on a shared bus; the MasterAnnouncer announces the winner.
decides Distance, capability, and workload are evaluated to select the best actor.
advises only The LLM can adjust the score within fixed limits; it never overrides the decision.

Future work

Where INTER.act is heading

▸ close the gap

Complete the recovery

  • Release locks on heartbeat loss so re-allocated actors finish delivery.
  • Faults recover end to end, not just re-allocation.
▸ truly contested

Beyond single-bidder

  • Move past the single eligible bidder toward genuinely distributed allocation.
  • More actors, real competition on the bus.
▸ UI · build it

Visual builder

  • Parallel routes and per-actor parameters authored from the canvas.
▸ beyond the warehouse

Any multi-agent system

  • The warehouse is one scenario - the coordination foundation is domain-agnostic.
  • Drone displays (one drone drops out), underwater cleaning fleets, defence and search-and-rescue swarms.
